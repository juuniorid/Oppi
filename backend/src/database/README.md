# Database

This folder contains all database-related logic, migrations, and seed data for Oppi.

## Structure

```
database/
├── schema.ts          # Drizzle ORM schema definitions
├── db.ts              # Database connection instance
├── init.ts            # Database initialization
├── migrations/        # Migration files (generated by Drizzle)
├── seeds/
│   └── seed.ts        # Initial seed data
└── scripts/
    ├── init.sh        # Initialize database (migrations + seed)
    └── reset.sh       # Drop and recreate database
```

## Setup

### 1. Configure Environment Variables

```bash
cp ../.env.example ../.env
# Edit .env with your database credentials
```

Example:
```env
DATABASE_URL=postgresql://oppi:oppi@localhost:5432/oppi
```

### 2. Initialize Database

From the root directory:

```bash
npm run db:init
```

Or manually:

```bash
npm run db:migrate  # Run Drizzle migrations
npm run db:seed     # Seed initial data
```

## Available Commands

Add these to `backend/package.json`:

```json
{
  "scripts": {
    "db:generate": "drizzle-kit generate:pg --schema=../../database/schema.ts",
    "db:migrate": "drizzle-kit migrate:pg --schema=../../database/schema.ts",
    "db:seed": "ts-node database/seeds/seed.ts",
    "db:init": "bash database/scripts/init.sh",
    "db:reset": "bash database/scripts/reset.sh"
  }
}
```

## Schema

The schema is defined in `schema.ts` using Drizzle ORM. Key tables:

- **users** - Teachers, parents, admins
- **groups** - Kindergarten classes/groups
- **children** - Children enrolled in groups
- **posts** - Announcements from teachers
- **messages** - Direct messages between users
- **parentsToChildren** - Many-to-many relationship (up to 2 parents per child)

See [docs/SCHEMA.md](../docs/SCHEMA.md) for detailed schema information.

## Migrations

Migrations are automatically generated by Drizzle Kit when you update the schema:

```bash
npm run db:generate  # Generate new migration
npm run db:migrate   # Apply pending migrations
```

## Seeding

The seed file creates:

- 2 kindergarten groups
- 3 users (1 teacher, 2 parents)
- 3 children
- Parent-child relationships

Run with:

```bash
npm run db:seed
```

## Database Reset

To drop and recreate the database:

```bash
npm run db:reset
npm run db:init
```

⚠️ **Warning**: This will delete all data!

## Development

When developing locally with Docker Compose, the database is initialized automatically
on first startup via healthchecks and depends_on conditions.
